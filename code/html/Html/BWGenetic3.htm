The Genetic Algorithm has succeeded in diverse problem spaces. It is particularly adept at solving problems that are traditionally intractable, such as high dimensional, multimodal, nonlinear problems subject to nonlinear constraints. For example, the class of NP-hard problems is amenable to the Genetic Algorithm. Because of its generality, the Genetic Algorithm can be modified to solve instances of these problems, and in some cases, it does so optimally. There are many instances of the Genetic Algorithm solving such problems, examples include, the Traveling Salesman Problem, and the Knapsack Problem among many others.

II. Design

[Purpose of Object Oriented Version]

In the last section, it was seen that the Genetic Algorithm is rather general. To apply the Genetic Algorithm to a specific problem case, the algorithm must be modified ad hoc to fit the environment of the problem space. Using the principles of object oriented programming, this modification to the particular problem space can be mitigated. Objects can be built that can be used in a ‘plug and play’ manner to implement the Genetic Algorithm for each particular problem.

(1) Initialize a population with random genes
(2) While the number of generations < required OR best fit individual is < required do:
        While current population size < required population size do:
            Select parents according to selection probability and fitness levels
            Crossover parents according to crossover probability to create offspring
            Mutate offspring according to mutation probability at each locus
(3) Return best fit individual.
