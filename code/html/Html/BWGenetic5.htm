[Goals of the Object Oriented Genetic Algorithm cont.]

‘setCrossoverStrategy()’: takes an object of type ‘CrossoverStrategy’ (defined shortly) as a parameter, ‘setMutationStrategy()’: takes as a parameter an object of type ‘MutationStrategy’ (defined shortly).

The interface ‘MutationStrategy’ declares the single method ‘mutate()’: which takes a parameter of type Chromosome and returns an object of type Chromosome.

The interface ‘SelectionStrategy’ declares the single method ‘getParent()’: which takes a Vector of Individuals as a parameter and returns a parent of type ‘Individual’.

There will, of course, be specific implementation classes that provide actual definitions for the various interfaces. Thus, the concrete class ‘RouletteWheelStrategy’ would inherit ‘SelectionStrategy’ and define a parent selection strategy based on a roulette wheel paradigm such that the more fit the individual is, the greater the chance that it would be selected.

The concrete classes ‘ValueEncodedChromosome’ and ‘BinaryEncodedChromosome’ would inherit the Chromosome abstract class and provide the details necessary for the implementation. The abstract class ‘CrossoverStrategy’ will have subclasses such as ‘SinglePointCrossoverStrategy’ and ‘TwoPointCrossoverStrategy’ that will define how the Chromosomes crossover during reproduction.

Finally, the interface ‘Individual’ will be derived by a particular instance of the problem that is to be solved. This class will be the only class that will change each time the problem changes, thus, this class can be plugged into the OOGA to derive a solution.
