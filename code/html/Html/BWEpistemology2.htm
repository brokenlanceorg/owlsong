Informally, we can see this by mapping the output symbols, graphics, and sounds to a unique prime number, then form the product of all the output--by the fundamental theorem of arithmatic, this number is unique. Thus, any program is equivalent to the printing out of a number and then halting.

Thus, our programs on M can be seen as algorithms that print out a number and then halt. We say that a given program P´, whose output is c, is elegant if for all programs P that output c it gives the minimal value for length(P).

It is clear that if program P is elegant, length(P) is the smallest number of keystrokes needed to program M to print out c and stop. For example, consider the program that prints out 

"123456789101112131415161718192021"

One way to code this is to simply type out 

"123456789101112131415161718192021"
on the keyboard. At that point, M has not been programed to do anything so it is stopped and the output is printed out on the screen as desired. Another way to program the output is by writing a program such as: 
"for(int i=1;i<22;i++) {print(i);}"
This program performs the same output and the length is the same as the previous program. So, from an information-theoretic point of view, the two programs above are equivalent. However, if we wanted our output to be 
"12345678910111213141516171819202122"
The second program could be modified to produce the desired result but the length would stay the same; thus, in this scenario, the second program is seen to be 'more elegant' than the first.
