[Turing Machines]

Turing machines, in essence, are highly abstracted idealizations of computers. They were invented by the mathematician Alan M. Turing in the mid 1930’s. He created his machines to answer questions about the foundation of mathematics, not to compute anything substantial. Specifically, he set out to solve Hilbert’s Entscheidungsproblem (the Decision Problem).

Basically, Hilbert wanted to know if there was an effective procedure to determine in advance if a particular theorem follows logically from a given set of axioms. The difficulty in answering this question was that the notion of “effective procedure”. To thoroughly define this idea, Turing invented his machines.

Essentially, a Turing machine consists of two components: an infinitely long tape sectioned off into squares that can contain either a zero or a one, and a scanning head device that can read a symbol off the tape and/or write a symbol to the tape and move the tape forward or backward one square.

The behavior of the read-write head is governed by an algorithm, or program, whose syntactical repertoire consists of the following:

  • PRINT 0
  • PRINT 1
  • GO LEFT 1 SQUARE
  • GO RIGHT 1 SQUARE
  • GO TO STEP I IF THE CURRENT SQUARE CONTAINS 0
  • GO TO STEP I IF THE CURRENT SQUARE CONTAINS 1
  • STOP

From these instructions, one can compose Turing-Post programs. Turing showed that if we allow the read-write head to contain a ‘state’, then we can eliminate the two GOTO statements above. Seems like a harmless machine, right?

Turing showed that anything that any computer can compute can be computed by an appropriate Turing machine.
