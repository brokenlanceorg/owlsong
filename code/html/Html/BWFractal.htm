When constructing a new algorithm to solve a particular problem, one must always consider the error analysis. That is, how fast does the purported algorithm converge to the solution set? Does the algorithm even converge to the solution set? If not, what classes of data sets engender difficulties? Questions such as these should be answered by an in-depth mathematical analysis of the algorithm. However, sometimes this analysis is difficult, not readily available, or impossible. For the previous two cases, it would be beneficial if the algorithm could be applied against a benchmark set (a data set chosen specifically for the problem at hand), and the output be arranged in such a way that a simple visual inspection could be performed to ascertain the effeciveness of the algorithm. In many instances, this representation is just what fractals perform. As an example, consider Newton's method for finding solution points of an equation. [ The stability and rate of convergence for Newton's method are well known, but the following method can be applied to other solution-finding algorithms equally well. ] Pick a point in the complex plane and multiply by its conjugate. Use this new real point as the initial point in Newton's method. Newton's method will then create a sequence of points (usually a Cauchy sequence) that will converge to the nearest solution point--provided that the initial approximation was chosen close enough to the real solution point.
When the algorithm finds the solution point, determine the cardinality of the sequence that the algorithm produced, and color-code the initial point to this number. Since the initial point was in the complex plane, this point can be mapped easily to a pixel on the computer screen to generate a visual representation of the effectiveness of the algorithm.
